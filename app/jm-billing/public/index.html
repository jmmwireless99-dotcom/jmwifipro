Dry-run safety mode

      
      
When ON, any command that would change the router (add/remove/set firewall, proxy, leases, VLANs, profiles) is logged but not sent. Reads still work. Use this to see exactly what the panel would do before letting it touch your live router. Turn OFF to apply for real.

      

      


      
Data maintenance (storage)

      
See what's taking space and archive old logs. Cleanup saves old records to a file first, then removes them from the live database (keeping the last N months). Your customers, payments, and invoices are never auto-cleared.

Multi-site routers 
      VPN → MikroTik API per site

    

      
One panel, many MikroTik sites

      
Each site is a MikroTik reached over VPN. Customers are tied to a site — suspend, renew, and provision always go to the correct router automatically.

      

        ☁ This panel→
        VPN tunnel→
        🖧 Site A→
        PPPoE clients
        +
        🖧 Site B→
        PPPoE clients
      

    

    

      📋 Multi-site setup guide (recommended for JM WIFI)
      

        
Use this layout: Panel (VPS) → VPN → each branch MikroTik. Customers must point at the right site before you provision.

        

          
1 · Add each siteClick + Connect a new MikroTik site. Enter VPN IP, API user, and password for each branch router.

          
2 · Test every sitePress Test on each card until it shows connected. Offline sites will break provision.

          
3 · Set default siteEdit your main/core router → check Default router. Use this only for unassigned customers.

          
4 · Name sites clearlye.g. Candelaria Main, PPPOE-3, Uson — match your barangay or tower.

          
5 · Add plans & customersCreate packages in Plans / Packages, then add clients under Customers and pick the Target site for each one.

          
6 · ProvisionClick prov after adding. If the account already exists on the router, the panel updates it (no duplicate error).

        

        
💡 Moving a client to another site? Use the 🖧 move button — it removes them from the old router and creates them on the new one.
